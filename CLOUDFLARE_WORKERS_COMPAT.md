## Partial Support (9 APIs)

⚠️ Limited implementation:

- **Context (ctx)** - Basic pattern supported via Promise pattern
- **Scheduled handlers** - scheduler.wait() replaceable by setTimeout
- **TLS/SSL** - Basic support via Deno TLS
- **EventSource** - One-way server updates only
- **DNS resolution** - Via public DNS API (not native)
- **Buffer (Node.js)** - Use Uint8Array alternative
- **Stream (Node.js)** - Use Web Streams API
- **events module (Node.js)** - Use EventTarget
- **zlib module (Node.js)** - Use CompressionStream

## Recommendations for Users

### For Most Use Cases
Use the **Web Standards APIs** - they have excellent support:
- fetch() for HTTP requests
- Web Streams for data processing
- TextEncoder/TextDecoder for encoding
- CompressionStream for compression
- EventTarget for events
- Web Crypto for cryptography

### For Node.js Code
**Migrate to Web Standards:**
- `fs` → Use fetch + external file service
- `path` → Use URL API
- `events` → Use EventTarget
- `crypto` → Use Web Crypto API
- `stream` → Use Web Streams API
- `Buffer` → Use Uint8Array
- `require()` → Use import/export

### For Advanced Features
**Consider Alternatives:**
- KV storage → Use Redis or external service
- Database → Use external DB via fetch
- WebSockets → Use Server-Sent Events or polling
- Scheduled tasks → Use external cron service
- HTML transformation → Use external service or regex
